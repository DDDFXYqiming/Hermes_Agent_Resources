# 数据验证清单

## 必须实时验证的数据

1. 当前股价、市值、PE-TTM、PB、ROE-TTM。
2. 最新年报/季报的 ROE、净利润、净资产、分红率。
3. ETF 跟踪指数、编制规则、成分股、费率、分红记录。
4. 13F、公司公告、回购、股息税、政策变化。
5. 债券收益率、商品价格、汇率等宏观输入。

## 来源优先级

优先使用权威一手或准一手资料：公司公告、年报/季报、交易所披露、基金公司资料、监管披露、官方税务/政策文件，以及可靠行情数据源。

不要在公开 skill 中硬编码具体本地路径、个人账号、临时检索链接或冗长网站清单。实际执行时按任务所在地和资产类别选择可访问的可靠来源，并在输出中写明数据日期与来源类型。

## 核验规则

- 同一关键数值尽量至少查 2 个来源；若只有一个来源，降低置信度。
- 区分作者原文数据与当前数据：原文数据可用于复盘，当前分析必须更新。
- 把日期写进输出，例如“YYYY-MM-DD PE-TTM = ...”。
- 不同口径冲突时说明：TTM/静态/扣非/摊薄/年化。
- 不得用搜索摘要里的数字直接替代公告原文，除非任务明确只需要粗略解释。

## 常见时效陷阱

- 分红政策会变化，修正 PR 时必须使用最新分红率。
- 管理层交接、13F 披露、ETF 编制规则等事件都有报告期和披露日差异。
- ETF 指数规则和成分权重会调整，必须查最新产品资料。
